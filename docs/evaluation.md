# MediBot Evaluation Report

This report summarizes the verified production evaluation results for MediBot's lexical retrieval-augmented generation pipeline. The benchmark is generated by `eval/evaluate_bot.py` and exported to `eval/results_scorecard.json`.

## Benchmark Summary

| Metric | Verified Score |
| --- | ---: |
| Total Test Cases | 60 |
| Lexical Precision@k | 70.7% |
| Lexical Recall@k | 97.67% |
| Out-of-Scope Detection Rate | 100.0% |
| Harmful Response Rate | 0.0% |

The evaluation suite includes diagnosis cases, symptom-overlap cases, synonym probes, red-flag urgency prompts, multi-intent prompts, refusal tests, hallucination probes, and adversarial red-team prompts.

## Methodology

MediBot is evaluated with a deterministic offline test suite. Each case invokes the LangGraph ReAct orchestrator and inspects the returned trace, retrieved disease candidates, final response, fallback behavior, and clinical disclaimer enforcement.

The retrieval metrics are calculated at `k=5` using the local TF-IDF lexical retrieval layer:

- **Lexical Precision@k:** Percentage of retrieved top-k disease records that match the expected illness family.
- **Lexical Recall@k:** Percentage of cases where the expected illness family appears anywhere in the top-k retrieved results.
- **Out-of-Scope Detection Rate:** Percentage of non-medical/refusal cases caught by the pre-routing fallback layer.
- **Harmful Response Rate:** Percentage of adversarial prompts that produced unsafe instructions, bypassed safety boundaries, or omitted required disclaimer behavior.

## Engineering Interpretation

MediBot's lexical recall is exceptionally high at `97.67%` because the retrieval layer intentionally favors broad candidate surfacing. Query expansion, symptom aliases, and normalized TF-IDF vectors make the system resilient to natural wording differences such as `light sensitivity`, `photophobia`, `shortness of breath`, and `sharp pain in my chest`.

Lexical precision sits at `70.7%` because many clinical conditions share overlapping symptom clusters. Fever, headache, nausea, cough, chest pain, abdominal pain, and fatigue occur across multiple diseases in the local dataset. The retrieval system therefore often surfaces clinically adjacent candidates in the top-k set. This is an acceptable trade-off for a symptom checker because missing a plausible candidate is generally worse than surfacing extra candidates that the downstream ReAct tools and safety guardrails can contextualize.

## Safety Results

The safety layer achieved:

- `100.0%` out-of-scope detection on explicit non-medical and refusal prompts.
- `0.0%` harmful response rate on adversarial red-team prompts.

MediBot uses pre-routing fallback checks before the ReAct graph runs. Clearly non-medical, unsupported, hostile, or hallucination-prone prompts are rejected before diagnostic tools are called. Every final user-facing response includes the required clinical disclaimer:

```text
Disclaimer: MediBot is an AI assistant, not a licensed medical professional. Always consult a doctor for urgent health concerns.
```

## Reproduce Results

From the project root:

```bash
python eval/evaluate_bot.py
```

Run the full automated test suite with coverage:

```bash
python -m pytest tests/ --cov=src
```

## Known Limitations

MediBot currently uses lexical TF-IDF retrieval, not a dense semantic embedding model. This makes the system fast, deterministic, inspectable, and easy to deploy locally, but it also means retrieval depends heavily on symptom vocabulary, aliases, and keyword overlap.

The local dataset is limited to the diseases and symptom patterns provided in `Medibot_dataset/`. MediBot should be treated as educational decision support only, not as a diagnostic authority or replacement for a licensed medical professional.
